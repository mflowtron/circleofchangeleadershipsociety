

# Add Sign in with Google

## Overview
Add Google Sign-in functionality to the authentication page, allowing users to sign in with their Google account in addition to the existing email/password method. This will use Lovable Cloud's managed Google OAuth solution, which requires no additional API keys or configuration.

## What You'll Get
- A "Continue with Google" button on the login page
- Seamless authentication flow with Google's OAuth
- Automatic profile creation for new Google sign-in users
- Works alongside existing email/password authentication

## Implementation Steps

### Step 1: Configure Google OAuth
Use the Lovable Cloud social auth configuration tool to enable Google OAuth and generate the required integration module.

### Step 2: Update Auth Page
Modify `src/pages/Auth.tsx` to add a Google sign-in button:
- Import the Lovable auth module
- Add a "Continue with Google" button with Google icon
- Create handler function that calls `lovable.auth.signInWithOAuth("google")`
- Add visual separator between social login and email/password forms
- Handle loading and error states

### Step 3: Handle OAuth Callback
The AuthContext already handles auth state changes via `onAuthStateChange`, so OAuth callbacks will be processed automatically. The existing `handle_new_user` database trigger will create profiles for new Google users.

## Technical Details

**Files to modify:**
- `src/pages/Auth.tsx` - Add Google sign-in button and handler

**Files to be generated:**
- `src/integrations/lovable/` - Auto-generated by the configuration tool

**Dependencies:**
- `@lovable.dev/cloud-auth-js` - Will be installed automatically

**UI Changes:**
- Google button placed above the email/password tabs
- Visual divider with "or" text separating social login from form login
- Consistent styling with existing button design
